# Техническое задание

## Проект: WEB-AGENT

## 1. Цель разработки

Разработка кроссплатформенного программного агента (web-agent), который взаимодействует с удалённым сервером через Web API для получения заданий, их выполнения и передачи результатов обратно на сервер.

Агент должен работать в автоматическом режиме и обеспечивать обмен данными с сервером по протоколу HTTP/HTTPS.

---

# 2. Область применения

Программный агент предназначен для:

* удалённого управления заданиями
* выполнения задач, получаемых с сервера
* передачи результатов выполнения на сервер
* автоматизации обработки заданий через Web API

---

# 3. Функциональные требования

## 3.1 Регистрация агента

Агент должен иметь возможность регистрации на сервере.

При регистрации выполняется HTTP запрос:

```id="regreq"
POST /app/webagent1/api/wa_register/
```

Сервер возвращает:

* код ответа
* сообщение
* уникальный `access_code`

Полученный `access_code` используется для дальнейшей авторизации агента.

---

## 3.2 Получение задач

Агент должен периодически запрашивать задачи у сервера.

Запрос выполняется:

```id="taskreq"
POST /app/webagent1/api/wa_task/
```

Сервер может вернуть:

* `WAIT` — задачи отсутствуют
* `RUN` — задача получена
* `session_id` — идентификатор сессии выполнения
* `task_code` — тип задачи

---

## 3.3 Выполнение команд

После получения задачи агент должен:

1. определить тип задания (`task_code`)
2. выполнить соответствующее действие
3. сформировать результат выполнения

Типы задач:

**TASK**

* выполнение стандартной команды
* результат передаётся как текст

**FILE**

* выполнение задачи с генерацией файла
* результат передаётся вместе с файлом

---

## 3.4 Передача результатов

После выполнения задания агент должен отправить результат на сервер.

Запрос выполняется:

```id="resultreq"
POST /app/webagent1/api/wa_result/
```

Передаваемые параметры:

| Поле        | Описание                             |
| ----------- | ------------------------------------ |
| result_code | код результата выполнения            |
| result      | JSON строка с результатом выполнения |
| file1       | файл (если требуется задачей)        |

Пример JSON результата:

```id="resultjson"
{
 "UID":"777",
 "access_code":"ACCESS_CODE",
 "message":"task completed",
 "files":1,
 "session_id":"SESSION_ID"
}
```

# 4. Нефункциональные требования

## 4.1 Сетевое взаимодействие

Программа должна поддерживать работу через:

* HTTP
* HTTPS

Используется клиентская библиотека HTTP.

---

## 4.2 Кроссплатформенность

Программа должна корректно работать на операционных системах:

* Linux
* Windows
* macOS

Сборка проекта должна осуществляться с использованием системы сборки **CMake**.

---

## 4.3 Производительность

Агент должен:

* потреблять минимальное количество системных ресурсов
* работать в фоновом режиме
* выполнять запросы к серверу с заданным интервалом

Рекомендуемый интервал опроса сервера: **5 секунд**.

---

# 5. Архитектура решения

Программа состоит из следующих компонентов:

| Компонент                   | Назначение                     |
| --------------------------- | ------------------------------ |
| HTTP-клиент                 | взаимодействие с серверным API |
| Обработчик задач            | анализ полученной задачи       |
| Исполнитель задач           | выполнение команды             |
| Модуль отправки результатов | передача результата выполнения |

Основной цикл работы агента:

1. отправка запроса на получение задачи
2. анализ ответа сервера
3. выполнение задания
4. отправка результата
5. ожидание следующего запроса

---

# 6. Используемые технологии

В проекте используются следующие технологии:

* язык программирования **C++**
* библиотека **cpp-httplib**
* библиотека **OpenSSL**
* система сборки **CMake**
* протокол передачи данных **HTTP/HTTPS**

---

# 7. Требования к запуску

Минимальные требования:

* компилятор C++ (GCC / Clang)
* библиотека OpenSSL
* CMake версии 3.10+

Сборка проекта:

```id="buildcmd"
mkdir build
cd build
cmake ..
make
```

Запуск программы:

```id="runcmd"
./web_agent
```

---

# 8. Ожидаемый результат

В результате выполнения проекта должен быть реализован рабочий программный агент, который:

* успешно регистрируется на сервере
* получает задания
* выполняет их
* отправляет результат выполнения
* корректно обрабатывает задания типов `TASK` и `FILE`
